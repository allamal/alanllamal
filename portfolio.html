📊 Dashboard inscripciones · Data Analytics

💡 El problema

Gestionar convocatorias a nivel nacional con miles de inscripciones y planillas ilegibles, métricas de conversión inexistentes, embudos de inscripción rotos y total ceguera respecto al impacto demográfico en tiempo real.

🚀 La solución

Es una plataforma de Business Intelligence a medida que transforma registros planos de Google Sheets en un centro de comando interactivo y de alta fidelidad visual. Diseñado específicamente para el monitoreo estratégico de proyectos sociales y educativos, este sistema permite auditar el pulso real de la convocatoria. Resuelve la falta de visibilidad automatizando el cálculo de tasas de conversión por canal, identificando brechas geográficas en tiempo real y segmentando audiencias de manera cruzada con un solo clic.

🌟 Características clave & Storytelling visual

Embudo de conversión realizado por canal; ranking total de provincias optimizado; filtros cruzados interactivos (Cross-Filtering); termómetro y análisis histórico (un módulo comparativo avanzado con cálculo de brechas que mide el rendimiento de la campaña actual frente al año anterior para asegurar el cumplimiento del objetivo global); acceso seguro y optimizado.

🛠️ Stack tecnológico

  • Frontend: HTML5, CSS3 (Bootstrap 5 para un diseño responsivo y moderno), JavaScript Moderno (ES6+).
  • Visualización de datos: Chart.js, configurado a bajo nivel para renderizados dinámicos, control de aspect-ratio adaptable y manejo de eventos onClick interactivos.
  • Backend & Base de datos: Google Apps Script (GAS), actuando como el motor del servidor, consumiendo y procesando datos de Google Sheets en milisegundos a través de ejecuciones asíncronas (google.script.run).

🎓 Gestión académica – Sistema de alerta temprana y Learning Analytics

💡 El problema

Los reportes tradicionales de los LMS entregan "fotos" estáticas del pasado, no flujos dinámicos del presente.

🚀 La solución

Es una plataforma web de Learning Analytics y retención estudiantil que transforma datos crudos de plataformas educativas en estrategias preventivas de intervención en tiempo real. A través de un ecosistema visual e intuitivo, la aplicación procesa automáticamente los reportes de actividad y avance para segmentar a los estudiantes según su nivel de riesgo. El sistema no solo calcula la inactividad cronológica, sino que evalúa el progreso académico relativo comparando a cada alumno con el ritmo real de su comisión, permitiendo a los tutores anticiparse a la deserción mediante un módulo CRM integrado para registrar bitácoras, gestionar alertas sistémicas y automatizar el seguimiento.

🌟 Características clave

  • Panel de Seguimiento Inteligente: Filtros dinámicos por carrera, comisiones, materias y buscador de alumnos por tokens que prioriza coincidencias exactas de identidad.
  • Algoritmo de alerta temprana: Clasificación automatizada en semáforos basada en reglas de negocio customizables de inactividad e ingresos nulos.
  • Detección de riesgo académico: Registro centralizado de gestiones; Módulo Business Intelligence (Datos macro) con matriz de análisis de salud institucional que permite comparar de manera longitudinal la evolución de la retención entre múltiples hitos de control.

🛠️ Tecnología y Arquitectura

Diseñado bajo un enfoque de costo cero de infraestructura ($0) y máxima eficiencia operativa, el proyecto se apoya por completo en soluciones Cloud corporativas:

  • Frontend: HTML5, CSS3, JavaScript Avanzado (ES6+) y Bootstrap 5.3 para una interfaz responsiva y fluida.
  • Backend: Google Apps Script (GAS) como entorno de ejecución del lado del servidor.
  • Persistencia de Datos: Google Sheets configurado como base de datos de alta velocidad mediante el uso de colecciones indexadas (Map en JS) para evitar cuellos de botella en consultas masivas.
  • Integración de Archivos: Google Drive API para la sincronización y consolidación automatizada de reportes Excel externos en lotes.
  • Seguridad: Autenticación nativa e inicio de sesión seguro mediante el protocolo OAuth2 de Google.

📈 El valor de negocio (El resultado)

Permite medir de manera real la tasa de efectividad de las intervenciones del equipo de retención, optimiza las horas de trabajo humano dirigiendo los esfuerzos hacia los estudiantes recuperables y reduce drásticamente las tasas de abandono institucional mediante el uso inteligente y pragmático de los datos.

📝 Evaluaciones sincrónicas

💡 El problema

La coordinación masiva de mesas examinadoras sincrónicas (finales virtuales) solía ser un caos logístico propicio para errores humanos: envío manual de enlaces de videoconferencia cruzados, planillas de notas internas vulnerables, retrasos en la validación contable de cuotas/documentación de alumnos y procesos de carga asincrónicos lentos en el sistema central.

🚀 La solución

Se desarrolló una WebApp robusta en Google Apps Script que actúa como un centro unificado de control para el ciclo completo de exámenes de carreras a distancia. El sistema automatiza de extremo a extremo la generación y comunicación de grillas de fechas, la vinculación segura de salas virtuales de Meet por mesa y el armado de actas dinámicas con filtros de bloqueo en tiempo real basados en el estado administrativo del alumno. Además, provee al docente de una interfaz simplificada donde asienta la calificación, sincroniza la información al instante con los sistemas centrales (Quinttos) y despacha las liquidaciones profesionales correspondientes de forma automática y transparente.

🌟 Características clave

  • Filtros Administrativos Automáticos: Bloquea o destaca dinámicamente en el acta interna a los alumnos con restricciones de documentación o cuotas pendientes antes de la evaluación.
  • Comunicación Segmentada y Automatizada: Despacha por correo los enlaces de Meet, listados de inscriptos personalizados y recordatorios con plantillas estructuradas tanto para docentes como para estudiantes.
  • Módulo Contable Integrado: Automatiza el cálculo y la asignación de honorarios docentes por mesa evaluadora con datos cruzados de administración institucional en un solo clic.
  • Auditoría de Tiempos Legales: Control y registro estricto en la base de datos de los tiempos de grabación de cada encuentro y las tolerancias legales de conexión para resguardo institucional.

🛠️ Stack tecnológico

  • Arquitectura del Servidor: Google Apps Script (WebApp) operando bajo lógica de inyección dinámica de HTML en plantillas (*HTML Templates*).
  • UI/UX Frontend: JavaScript Nativo (Manipulación avanzada de DOM) integrado con componentes responsivos de Bootstrap 5 y FontAwesome para control estético.
  • Integraciones API Core: Google Calendar API (Calendarización y control de salas Meet), Gmail API (Estructura automatizada de mails tipo), Drive API (Persistencia documental e historial de actas fijas).

📈 Impacto Operativo

Elimina el 100% de la fricción administrativa en periodos críticos de exámenes. Centraliza los datos macro para auditoría académica inmediata y transforma un flujo operativo manual que tomaba días enteros de trabajo humano coordinado en un proceso sistémico automatizado, seguro y sin fallas.
